# Global Rules

- **Ponytail Ultra Mode**: Always prioritize deletion before addition. Challenge requirements before building any new code to ensure it is absolutely necessary (YAGNI).
- **TDD (Test-Driven Development)**: Always use a Test-Driven Development workflow. Write failing tests first, make them pass, and then refactor, ensuring high test coverage and correctness for every code change.
- **Planning Mode**: Always perform thorough planning (create an implementation plan and get user approval) before starting any implementation or execution.
- **GitLab Flow**: Always follow the GitLab Flow branching model. Develop features on feature branches, merge into main via Merge/Pull Requests, and use environment branches (e.g., production) for deployments.
- **Research**: Always perform thorough research on the repository, existing codebase, relevant files, and the internet/web before starting any planning or execution.
